Current file snapshot to anchor the work
- The project is now built around the expanded 1999-2019 draft cohort rather than the old 2005-2019 version.
- The player feature table and the PCA + K-means outputs already exist and should be reused rather than recomputed unless the feature-engineering logic changes.
- The clustering stage was run on the full player feature table without the 40-game filter. That choice should remain explicit in the write-up.
- The downstream Years 5-7 target sample and Year-5 salary sample will still shrink after merges and target-specific filters.

Important project constraints
- The rookie-contract game logs are traditional boxscore logs, not advanced tracking data.
- The current target file does not contain Win Shares, BPM, VORP, RAPM, or other advanced all-in-one value metrics.
- Advanced metrics may only be used if they can be derived from existing columns or if a later step explicitly adds a new source.
- Hoopshype player IDs do not directly align with NBA API IDs, so salary matching should be handled carefully, likely via normalized player-name plus season matching.
- Keep the analysis honest about sample shrinkage after filters and merges.
- Do not assume all 1999-2019 draftees have full Years 5-7 realized outcomes. The target file reflects only observed NBA seasons, so players who wash out early will create structurally missing later-career rows rather than explicit zero seasons.

Overall objective
Build a post-rookie forecasting system that uses:
1. Seasons 1-4 game-by-game trajectories,
2. early-career archetype information,
3. and a core ShotChartDetail-based shot-style representation pipeline

to predict:
1. later-career performance over Years 5-7 using supported boxscore-based outcomes,
2. Year-5 contract value as salary-cap percentage,
3. a categorical outcome label: sleeper, neutral, or bust.

The project must also assemble three stakeholder-facing final deliverables:
1. Performance Forecast
2. Player Archetype & Comparable Profile
3. Salary Decision Support

Recommended modeling structure
- Interpretable aggregate baseline models first.
- PCA + K-means clustering already completed and should now be integrated rather than treated as optional side analysis.
- ShotChartDetail-based CNN autoencoder pipeline is now core scope, especially for Deliverable 2.
- Then an LSTM hybrid model using the full Seasons 1-4 game path together with clustering predictors and, if validated, player-level shot-style embeddings.
- Prefer a multi-task setup with separate regression and classification heads, but split into separate models if missingness or sample coverage makes joint training unstable.

   - Merge `salaries_players_1990_2025_combined.csv` with `Cleaned_Salary_Cap.csv`

   Recommended approach:
   - Build an early-career expectation score from Seasons 1-4 aggregate performance and opportunity
